# STEP-808

A four-track drum sequencer in the browser. React, TypeScript, Web Audio API.

Built as a portfolio piece at the intersection of frontend engineering and audio — three of the four voices (kick, hi-hat, clap) are synthesised live with the Web Audio API; the snare is a small WAV sample. Patterns survive a page reload and can be shared via URL hash, so the link itself plays back what you made.

This app lives inside the `stefanvesper-com-new` repo and is served as a subpath of the main Astro site at **<https://stefanvesper.com/drums/>**.

## Run locally

```bash
cd drums-app
npm install
npm run dev
```

Then open the URL Vite prints (defaults to `http://localhost:5173/drums/`).

## Build + deploy to the website

One command builds the app and copies the output into the Astro site's `public/drums/` folder:

```bash
cd drums-app
npm run deploy
```

After that, the static assets sit in `../public/drums/`. Commit and push from the website root:

```bash
cd ..
git add public/drums drums-app
git commit -m "Update drums app"
git push
```

GitHub Actions deploys the website to the Hetzner server automatically, so the new build is live within a minute or two at <https://stefanvesper.com/drums/>.

## How the audio works

The scheduler follows Chris Wilson's lookahead pattern from [A Tale of Two Clocks](https://www.html5rocks.com/en/tutorials/audio/scheduling/):

- A `setInterval` ticks every 25 ms.
- On each tick the engine looks 100 ms into the future and schedules every step that falls within that window with sample-accurate Web Audio time.
- A separate `requestAnimationFrame` loop polls `AudioContext.currentTime` and drives the visual cursor — so the playhead lights up exactly when the sound plays, even if the main thread is busy.

This is the standard pattern professional in-browser audio uses (Soundtrap, Splice's web tools, web-based Ableton clones).

### Voices

| Voice  | Source                                                                |
| ------ | --------------------------------------------------------------------- |
| Kick   | Sine oscillator, frequency sweep 150 → 45 Hz, plus a brief noise click |
| Snare  | `/public/snare.wav` (synthesised at build time — see below)           |
| Hi-Hat | White-noise burst through highpass + bandpass filters                 |
| Clap   | Three short noise bursts plus a longer tail, all bandpass-filtered    |

### Replacing the snare

Drop your own 16-bit PCM WAV at `public/snare.wav` — anything under ~250 ms works well. The included sample is generated by a small Python script (`scripts/build_snare.py` if you want to regenerate it) that mixes filtered noise with a 200 Hz tonal "ping" that decays in pitch.

## URL pattern sharing

The whole pattern + BPM is encoded into the URL hash:

```
.../#p=A1B2C3D4E5F60000&bpm=124
```

Each step is one hex nibble; bit 0 = kick, bit 1 = snare, bit 2 = hat, bit 3 = clap. Open the link and the sequencer boots into that exact pattern. Hit **SHARE** to copy the current URL to your clipboard.

## Keyboard

- **Space** — play / stop
- Click a voice label to preview that sound
